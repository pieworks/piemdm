# 审批管理

审批管理是 PieMDM 数据质量保障的核心环节。通过配置化的审批流，系统可以确保关键数据的变更经过相关责任人的审核。

## 1. 审批中心概览

用户可以通过导航栏的“审批管理”进入审批中心，这里集成了所有与当前用户相关的审批任务，主要分为以下三个维度：

- **待我审批**：当前需要您处理的审批任务。
- **我已处理**：您已经处理完毕的历史审批记录。
- **我的申请**：您本人发起的所有数据变更申请及其当前进度。

## 2. 处理审批任务

当有新的申请需要您审批时，您可以按照以下步骤操作：

1. **查看列表**：在“待我审批”页签中，系统展示了申请标题、发起人、申请时间、当前节点等信息。
2. **详情查看**：点击“详情”进入任务处理页面。
3. **核对表单**：页面会展示本次变更的详细数据。如果是修改操作，系统通常会高亮显示变更的字段。
4. **历史溯源**：在页面下方可以查看该申请之前的处理轨迹和意见。
5. **执行操作**：
   - **同意 (Approve)**：数据流向下一个节点，或最终生效。
   - **驳回 (Reject)**：申请被退回给发起人，发起人需要修改后重新提交。
   - **意见填写**：无论执行何种操作，都建议填写具体的处理意见。

## 3. 发起人的后续操作

作为申请人，您可以在“我的申请”中实时追踪进度：

- **追踪状态**：查看申请目前停留在哪个审批节点，以及谁是当前的审批人。
- **撤回申请**：在流程结束前，您可以撤回已提交的申请（视权限配置而定）。
- **流程结束**：
  - **审批通过**：数据会自动从 `_draft` 表迁移至正式表，并更新版本记录。
  - **审批驳回**：您可以点击“再次提交”，基于原有的申请数据进行修改后，重新发起审批。

## 4. 审批与数据生效逻辑

<callout emoji="⚙️" background-color="pale-gray">
系统后台会自动关联业务表与审批定义。当您在“实体管理”中点击提交时，后台会检测该表是否配置了强制审批。如果是，数据会被插入到该表的 <code>_draft</code> 后缀表中。只有流程标记为 <code>Approved</code> 后，系统才会执行 <code>MOVE TARGET</code> 操作，将草稿覆盖或合并到正式数据中。
</callout>

## 5. 常见问题

<callout emoji="❓" background-color="light-purple">
审批中发现数据有错，可以直接修改吗？
</callout>

审批人通常只有查看和决定权，不能直接修改他人的申请数据。如果您认为数据有误，请通过“驳回”并附注修改建议，让发起人自行修正。

<callout emoji="❓" background-color="light-purple">
如何查看历史审批记录？
</callout>

您可以查阅“我已处理”或者在特定的实体详情页查看其“变更日志 (LOG)”，日志中详细记录了每一次审核通过后的数据版本。
