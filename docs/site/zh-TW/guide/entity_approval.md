# 審批管理

審批管理是 PieMDM 數據質量保障的核心環節。通過配置化的審批流，系統可以確保關鍵數據的變更經過相關責任人的審核。

## 1. 審批中心概覽

用戶可以通過導航欄的“審批管理”進入審批中心，這裡集成了所有與當前用戶相關的審批任務，主要分為以下三個維度：

- **待我審批**：當前需要您處理的審批任務。
- **我已處理**：您已經處理完畢的歷史審批記錄。
- **我的申請**：您本人發起的所有數據變更申請及其當前進度。

## 2. 處理審批任務

當有新的申請需要您審批時，您可以按照以下步驟操作：

1. **查看列表**：在“待我審批”頁籤中，系統展示了申請標題、發起人、申請時間、當前節點等信息。
2. **詳情查看**：點擊“詳情”進入任務處理頁面。
3. **核對表單**：頁面會展示本次變更的詳細數據。如果是修改操作，系統通常會高亮顯示變更的字段。
4. **歷史溯源**：在頁面下方可以查看該申請之前的處理軌跡和意見。
5. **執行操作**：
   - **同意 (Approve)**：數據流向下一個節點，或最終生效。
   - **駁回 (Reject)**：申請被退回給發起人，發起人需要修改後重新提交。
   - **意見填寫**：無論執行何種操作，都建議填寫具體的處理意見。

## 3. 發起人的後續操作

作為申請人，您可以在“我的申請”中實時追蹤進度：

- **追蹤狀態**：查看申請目前停留在哪個審批節點，以及誰是當前的審批人。
- **撤回申請**：在流程結束前，您可以撤回已提交的申請（視權限配置而定）。
- **流程結束**：
  - **審批通過**：數據會自動從 `_draft` 表遷移至正式表，並更新版本記錄。
  - **審批駁回**：您可以點擊“再次提交”，基於原有的申請數據進行修改後，重新發起審批。

## 4. 審批與數據生效邏輯

<callout emoji="⚙️" background-color="pale-gray">
系統後臺會自動關聯業務表與審批定義。當您在“實體管理”中點擊提交時，後臺會檢測該表是否配置了強制審批。如果是，數據會被插入到該表的 <code>_draft</code> 後綴表中。只有流程標記為 <code>Approved</code> 後，系統才會執行 <code>MOVE TARGET</code> 操作，將草稿覆蓋或合併到正式數據中。
</callout>

## 5. 常見問題

<callout emoji="❓" background-color="light-purple">
審批中發現數據有錯，可以直接修改嗎？
</callout>

審批人通常只有查看和決定權，不能直接修改他人的申請數據。如果您認為數據有誤，請通過“駁回”並附註修改建議，讓發起人自行修正。

<callout emoji="❓" background-color="light-purple">
如何查看歷史審批記錄？
</callout>

您可以查閱“我已處理”或者在特定的實體詳情頁查看其“變更日誌 (LOG)”，日誌中詳細記錄了每一次審核通過後的數據版本。
